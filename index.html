PokeDex App Details

This is a JavaScript application that fetches data from an external API and displays it in a user-friendly manner. It utilizes HTML, Bootstrap CSS, and JavaScript to create a dynamic web interface for viewing the data.

I was the developer for this project.

Below were the technical requirements of the project:-
  • The app must load data from an external API; for instance, the Pokémon API.
  • The app must display a list of items loaded from that API after the page is loaded.
  • The app must enable the viewing of more details for a given list item on demand, such as when clicking on a list item.
  • The app must have CSS styling.
  • The JavaScript code must be formatted according to ESLint rules.
  • The JavaScript code may be manually formatted or by Prettier.
  • The app must use at least one additional complex UI pattern, such as a modal, for details or touch interactions.
  • The app may allow searching for items (e.g., searching for Pokémon).
  • The app must not throw any errors when being used.
  • The app should be deployed to a publicly accessible platform like GitHub Pages.
  • The app must work in Chrome, Firefox, Safari, Edge, and Internet Explorer 11.
Some things I did to improve the project:-
  • Used IIFE to avoid polluting the global namespace and used it in module pattern to learn JS coding best practices.
  • Used Bootstrap to make CSS implementation easier, responsive and accessible.
  • Used arrow functions to improve readability and code control.

Though this project turned out quite good I wish I used Jquery instead of vanilla javascript. Jquery makes coding much simpler and neater. Though it has performance issues with bigger apps, for smaller apps like this its not an issue.

Some of the things I learned while doing this project are- IIFE, DOM interaction, AJAX, CSS frameworks, complex UI patterns, Jquery, JavaScript performance improvement and debugging.

Technologies used in this project are HTML, Bootstrap CSS and Javascript.

MyFlix API Details

This is the server-side component of a movies database web application. This simple CRUD (Create, Read, Update, Delete) API application will provide users with access to information about different movies, directors and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

I was the developer for this project.

Below were the technical requirements of the project-
  • Return a list of ALL movies to the user
  • Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user
  • Return data about a genre (description) by name/title (e.g., “Thriller”)
  • Return data about a director (bio, birth year, death year) by name
  • Allow new users to register
  • Allow users to update their user info (username, password, email, date of birth)
  • Allow users to add a movie to their list of favorites
  • Allow users to remove a movie from their list of favorites
  • Allow existing users to deregister

Although the project only required written documentation in HTML, I took the initiative to create swagger UI for documentation. All the endpoints definition and data requirements can be easily seen and tested in the swagger UI.

Some of the things I learned while doing this project are- NodeJS package management, express, CORS implementation, database management & operations, authentication & authorization for JavaScript apps, Data Security, Validation & Ethics and swagger implementation.

Technologies used in this project are NodeJS, Express, MongoDB and Swagger.

MyFlix Client Details

This is the client-side component of a movies database web application. This simple React application will provide users with access to a view of list movies, list of genres & suggested movies from them, list of directors & suggested movies from them and profile view with ability to edit personal information, view users favorite movies list, delete user account.

I was the developer for this project.

Below were the technical requirements of the project- Main view
  • Returns ALL movies to the user (each movie item with an image, title, and description)
  • Filtering the list of movies with a “search” feature
  • Ability to select a movie for more details
  • Ability to log out

Single Movie view

  • Allows users to add a movie to their list of favorites

Signup view

Profile view
  • Displays user registration details

As seen above genre view and director view weren't a requirement for this project but I added them to make the app more engaging for the user.

Meet App Details

This is a meet app, displaying a list of upcoming events for a city and time of the events. It will also be available for users to use while offline. It is a serverless, progressive web application (PWA) with React using a test-driven development (TDD) technique. The application uses the Google Calendar API to fetch upcoming events.

I was the developer for this project.

Below were the technical requirements of the project-
  • The app must be a React application.
  • The app must be built using the TDD technique.
  • The app must use the Google Calendar API and OAuth2 authentication flow.
  • The app must use serverless functions (AWS lambda is preferred) for the authorization server instead of using a traditional server.
  • The app’s code must be hosted in a Git repository on GitHub.
  • The app must work on the latest versions of Chrome, Firefox, Safari, Edge, and Opera, as well as on IE11.
  • The app must display well on all screen sizes (including mobile and tablet) widths of 1920px and 320px.
  • The app must pass Lighthouse’s PWA checklist.
  • The app must work offline or in slow network conditions with the help of a service worker.
  • Users may be able to install the app on desktop and add the app to their home screen on mobile.
  • The app must be deployed on GitHub Pages.
  • The app must implement an alert system using an OOP approach to show information to the user.
  • The app must make use of data visualization.
  • The app must be covered by tests with a coverage rate >= 90%.
  • The app must be monitored using an online performance monitoring tool.

This app was built using TDD approach and it is PWA. The tools that were used build are Bootstrap, React, AWS lambda, Google Calendar API & OAuth2. Recharts for charting and Jest, Cucumber for testing.

Chat App Details

Developed using Expo and React Native TypeScript, this chat application seamlessly integrates Google Firebase for cloud storage and offers offline chat access. Utilizing the Gifted Chat library, it enhances user experience by allowing not only text messaging but also the capability to take and send photos, pick images from the library, and share geolocation.

I was the developer for this project.

Below were the technical requirements of the project-
  • The app must be written in React Native.
  • The app must be developed using Expo.
  • The app must be styled according to the given screen design.
  • Chat conversations must be stored in Google Firestore Database.
  • The app must authenticate users anonymously via Google Firebase authentication.
  • Chat conversations must be stored locally.
  • The app must let users pick and send images from the phone’s image library.
  • The app must let users take pictures with the device’s camera app, and send them.
  • The app must store images in Firebase Cloud Storage.
  • The app must be able to read the user’s location data.
  • Location data must be sent via the chat in a map view.
  • The chat interface and functionality must be created using the Gifted Chat library.
  • The app’s codebase must contain comments.

In developing the chat application, I opted for TypeScript to enhance code reliability and maintainability through its static type checking. This choice not only improved data integrity for the real-time communication features but also aligned with my goal of mastering industry-preferred technologies for large-scale application development.

Some of the things I learned while doing this project are- Expo, Android studio, React Native, TypeScript, Google Firebase, Gifted Chat library, Offline Support with AsyncStorage, Location Services, Image Handling, React Navigation, Real-Time Data Syncing and Listening, Styling and UI Design, Error Handling and Debugging.

Technologies used in this project are Expo, Android studio, React Native, TypeScript, Google Firebase, Gifted Chat library
